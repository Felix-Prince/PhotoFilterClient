# v0.1.0 — 阶段一·手动快筛

**首个可用版本。** 不接 AI，纯键盘操作快速筛选图片与视频，操作即文件移动。

> 即使没有后续 AI 阶段，这个版本也已经可用——这是工具的底线价值。

---

## ✨ 新功能

### 📁 图片/视频统一筛选
- 选目录后选择筛选类型（🖼 图片 / 🎬 视频），各自独立分类
- 容器隔离：图片进 `图片/{保留,删除,待定}`，视频进 `视频/{保留,删除,待定}`
- 同一目录下图片和视频互不干扰

### ⌨️ 键盘快筛（核心交互）
| 快捷键 | 动作 |
|--------|------|
| `P` | 保留 |
| `X` | 删除 |
| `U` | 待定 |
| `←` / `→` / `空格` | 切换照片 |
| `Tab` | 跳到下一张待处理 |
| `Enter` | 预览大图（系统查看器） |
| `Ctrl+Z` | 撤销 |
| 双击卡片 | 预览大图 |

### 📐 三区域布局
- **待处理区**（上方主区域）：未分类照片，按系统建议排序
- **保留 / 待定 / 删除 三分类区**（下方并排）：实时显示已分类项 + 计数
- 操作动画：照片从上方"飞"到下方对应区域

### 🖼 缩略图懒加载
- Rust 异步生成缩略图（`spawn_blocking`，不冻结 UI）
- IntersectionObserver 懒加载，仅可视区域请求
- 3 路并发控制，避免 IPC 过载
- 缩略图缓存到 `~/.cache/snappick-spike3/thumbnails/`，二次加载秒级

### 🎬 视频条件抽帧
- 启动时检测系统 `ffmpeg`
- 有 ffmpeg：调用 `ffmpeg -ss 1 -frames:v 1` 抽首帧做缩略图
- 无 ffmpeg：视频卡片显示 🎬 图标 + 文件大小，双击仍可用系统播放器预览
- 顶栏显示「⚠ 无 ffmpeg」警告提示

### 🔍 系统查看器预览
- 双击图片卡片 → 系统默认图片查看器
- 双击视频卡片 → 系统默认视频播放器
- 不在工具内实现放大，减少复杂度（设计文档 §3.5）

### ↩️ 撤销栈
- `Ctrl+Z` 撤销最近 50 步移动操作
- 后端返回 `MoveRecord`，撤销时反向 rename
- 撤销失败自动把记录放回栈，不丢失
- 仅当前会话有效，依靠「删除文件夹不自动清空」兜底

### 🔁 断点续传
- 扫描时根据所在子文件夹恢复状态（保留/删除/待定）
- 关软件重开，之前 P/X/U 的分类全部恢复
- 焦点自动落在第一张待处理照片

### 📦 配对原子移动
- JPG + RAW + XMP 视为事务单元，整体移动
- 任一文件失败 → 已移动项回滚 → 整体报错
- 修复了 Windows 大小写扩展名导致同一文件被收集两次的 bug

## 📊 Spike 阶段验证结论

开工前用 3 个 Spike 验证了关键不确定性：

| Spike | 假设 | 实测 | 结论 |
|-------|------|------|------|
| 1 | gemma4:e4b 可做轻量预判 | 63 张同场景照片 100% 输出 keep | ❌ 取消轻量层，全量走 InternVL3.5 |
| 2 | InternVL3.5 评分稳定 | 30 张 × 3 次一致率 100% | ✅ 稳定性约束有效 |
| 3 | Tauri 跑缩略图流畅 | 异步 + 逐张 + 3 路并发稳定 | ✅ Tauri v2 通过 |

## ⚠️ 已知限制

- **撤销栈仅当前会话有效**：关闭软件后无法撤销，但可从删除文件夹手动恢复
- **跨磁盘移动非秒级**：工作目录在 NAS/移动硬盘时，move 退化为复制+删除
- **视频无时长显示**：依赖 ffprobe，本版本未实现
- **视频不配对字幕**：.srt 等不跟随移动
- **无 AI 评估**：本版本纯手动，AI 审美评估在阶段三

## 📥 安装与运行

### 环境要求
- Node.js ≥ 18
- Rust ≥ 1.77
- ffmpeg（可选，视频抽帧需要）

### 从源码运行
```bash
cd snappick
npm install
npm run tauri:dev
```

### 打包
```bash
cd snappick
npm run tauri:build
```
